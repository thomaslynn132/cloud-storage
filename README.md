# Cloud File Hosting Platform

A scalable cloud file hosting platform with ad-supported download model and hybrid storage lifecycle system. Built as a content distribution system with aggressive abuse protection and cost management.

## 🧠 Product Overview

This platform serves as a **content distribution system** that allows:

* **Uploaders** to upload and share files globally
* **Free uploads** expire after 30 days (cost optimization)
* **Paid users** can store files permanently with annual subscription
* **Downloaders** access files via public links with mandatory ad viewing
* **Aggressive abuse protection** with hash blocking, virus scanning, and rate limiting
* **Cost-efficient operations** using Cloudflare R2 (no egress fees)

## 🚀 Quick Start

### Prerequisites

* Node.js 20+
* PostgreSQL 14+
* Redis 6+
* Cloudflare R2 account (or AWS S3 for testing)
* Stripe account (for payments)

### Backend Setup

1. **Clone and install dependencies:**
   ```bash
   cd backend
   npm install
   ```

2. **Configure environment variables:**
   ```bash
   cp .env.example .env
   # Edit .env with your credentials
   ```

3. **Set up PostgreSQL:**
   ```bash
   createdb filehost
   # Database schema auto-creates on first run (synchronize: true)
   ```

4. **Start Redis:**
   ```bash
   redis-server
   ```

5. **Run the backend:**
   ```bash
   npm run start:dev
   ```

6. **Access Swagger documentation:**
   ```
   http://localhost:3001/api/docs
   ```

### Frontend Setup

1. **Install dependencies:**
   ```bash
   cd frontend
   npm install
   ```

2. **Configure environment:**
   Create `.env.local`:
   ```
   NEXT_PUBLIC_API_URL=http://localhost:3001
   ```

3. **Run the frontend:**
   ```bash
   npm run dev
   ```

4. **Access the application:**
   ```
   http://localhost:3000
   ```

## 🔐 Core Features

### 1. Authentication & Users
* Email/password registration and login
* JWT access tokens (15 min) + refresh tokens (7 days)
* User roles: `free`, `paid`, `admin`
* Admin account auto-seeded on first run

### 2. File Upload System
* **Chunked & resumable uploads** (S3 multipart)
* **Direct-to-R2 uploads** (zero backend bandwidth)
* Pre-signed upload URLs generated by backend
* **Deduplication**: SHA256 hash checking reuses storage
* File metadata stored in PostgreSQL
* Storage quotas enforced by plan type

### 3. Content Distribution & Download
* Public shareable links
* **Ad-gated downloads**: Users must view ads before downloading
* Signed download tokens (prevents bypassing ad gate)
* CDN-ready with proper cache headers
* Rate limiting via Redis (prevents abuse)
* CAPTCHA integration ready
* Download count tracking per file

### 4. Storage Lifecycle Management
* **Free files**: Auto-delete after 30 days
* **Paid files**: Permanent until subscription expires
* Background cron jobs (BullMQ):
  * Daily cleanup of expired files
  * Subscription expiry checking
  * Grace period before deletion (3-7 days)
* Instant storage quota updates on delete

### 5. Aggressive Abuse Protection
* **Virus scanning**: ClamAV integration (buffer/stream scanning)
* **File type validation**: Blocked extensions and MIME types
* **Hash-based blocking**: Known illegal file hashes blocked
* **IP-based rate limiting**: Temporal IP blocks for abuse
* **DMCA/report system**: File reporting with admin review
* **Size limits**: 1GB free, 10GB paid per file
* **Request throttling**: Global and per-endpoint rate limits

### 6. Cost Management
* **Zero egress fees** with Cloudflare R2
* **Storage cost tracking**: $0.015/GB/month
* **Deduplication savings**: Identical files don't double storage cost
* **Projected monthly cost** reporting for admins
* **Direct uploads**: No backend bandwidth costs
* **CDN integration**: Cloudflare for global distribution

### 7. Payments System
* **Stripe integration** for subscription management
* **Annual subscription plans**:
  * Basic: 50GB storage (~$49/year)
  * Pro: 500GB storage (~$99/year)
* Webhook handling for subscription lifecycle
* Automatic plan upgrades/downgrades

### 8. Ads System (Core Revenue)
* **Mandatory ad gate** before every download
* Token-based verification (prevents bypassing)
* **Analytics tracking**:
  * Impressions
  * Clicks
  * Click-through rate (CTR)
  * Revenue per file (integration ready)
* Anti-bot measures (IP tracking, token validation)

## 🗄️ Database Schema

### Users Table
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR UNIQUE NOT NULL,
  password_hash VARCHAR NOT NULL,
  plan_type VARCHAR DEFAULT 'free',
  storage_used BIGINT DEFAULT 0,
  subscription_expires_at TIMESTAMP,
  is_admin BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Files Table
```sql
CREATE TABLE files (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  file_name VARCHAR NOT NULL,
  size BIGINT NOT NULL,
  hash VARCHAR NOT NULL,
  storage_key VARCHAR NOT NULL,
  is_permanent BOOLEAN DEFAULT FALSE,
  expiry_date TIMESTAMP,
  download_count INTEGER DEFAULT 0,
  is_deleted BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Downloads Table
```sql
CREATE TABLE downloads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  file_id UUID REFERENCES files(id),
  ip_address VARCHAR NOT NULL,
  ad_shown BOOLEAN DEFAULT FALSE,
  ad_clicked BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Payments Table
```sql
CREATE TABLE payments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  amount INTEGER NOT NULL,
  plan VARCHAR NOT NULL,
  stripe_subscription_id VARCHAR,
  expires_at TIMESTAMP NOT NULL,
  status VARCHAR DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW()
);
```

## ⚡ API Endpoints

### Authentication
```
POST   /auth/register           # Register new user
POST   /auth/login              # Login user
POST   /auth/refresh            # Refresh access token
```

### Upload
```
POST   /upload/init              # Initialize chunked upload
POST   /upload/complete         # Complete upload
```

### Files
```
GET    /files                    # List user files
GET    /files/:id                # Get file info
DELETE /files/:id                # Delete file
```

### Download (Ad-Gated)
```
GET    /files/:id/ad             # Get file info for ad display
POST   /files/:id/verify-ad      # Verify ad view, get download token
GET    /files/:id/download?token=...  # Download file (requires token)
```

### Payments
```
POST   /payments/subscribe       # Create subscription
POST   /payments/webhook         # Stripe webhook
```

### Ads
```
GET    /ads/stats?fileId=...    # Get ad statistics
```

### Admin
```
GET    /admin/stats              # Admin dashboard stats
```

**Full API documentation available at:** `http://localhost:3001/api/docs`

## 🔒 Security Features

1. **All uploads/downloads bypass backend** - Direct R2 URLs

4. **Rate limiting** - Per-IP and per-user throttling

6. **Virus scanning** - Scan before storing

8. **DMCA system** - Report abusive content

## 📊 Cost Optimization Strategies

* **Cloudflare R2**: No egress fees (vs S3's $0.09/GB)
* **Direct uploads**: Zero backend bandwidth
* **Deduplication**: Identical files share storage
* **Auto-expiry**: Free files deleted after 30 days
* **Lifecycle policies**: Automated cleanup jobs
* **CDN caching**: Reduce origin requests
* **Preview optimization**: Stream media, don't download
* **Compression**: Reduce storage footprint
